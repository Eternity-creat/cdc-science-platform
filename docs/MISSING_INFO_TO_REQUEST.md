# 需要向其他同学一次性确认的信息

更新时间：2026-07-02

## 向前端同学确认

这些内容本地后端代码无法完全推断，需要前端提供真实页面使用情况。

1. 页面清单
   - 文章列表页
   - 创建文章页
   - 大纲编辑页
   - 正文编辑页
   - 历史版本页
   - 配图管理页
   - Wiki 管理页
   - 模板管理页
   - LLM 配置页

2. 每个页面实际调用哪些接口
   - 接口路径
   - GET/POST/PUT/DELETE
   - 调用时机
   - 是否并发调用

3. 每个接口前端实际传什么参数
   - 必填字段
   - 可空字段
   - 默认值
   - 字段类型

4. 每个接口前端需要哪些返回字段
   - 列表页字段
   - 详情页字段
   - 编辑页字段
   - 状态字段如何展示

5. 空数据和异常展示规则
   - 无疾病/疫苗时怎么展示
   - 无人群/场景时是否允许创建
   - 无模板时是否允许创建
   - 文章不存在时怎么提示
   - Agent 不在线时怎么提示
   - `code=500` 但 HTTP 200 时前端是否统一处理

6. 当前报错材料
   - Network 请求截图
   - 请求体
   - 响应体
   - 控制台报错

## 向 Agent/研究生确认

这些内容本地能看到大体结构，但需要确认运行配置和失败格式。

1. Agent 服务端口和地址
   - 当前默认看到的是 `http://localhost:8001`

2. 真实模型配置
   - 文本生成模型
   - embedding 模型
   - 图片生成模型
   - API Key 由环境变量还是 `cdc_llm_config` 管理

3. Agent 失败返回格式
   - LLM 超时
   - API Key 错误
   - 模型返回空内容
   - 解析意图失败

4. 是否需要真实跑通以下 Agent 接口
   - `/api/agent/parse-intent`
   - `/api/agent/generate`
   - `/api/agent/retrieve`
   - `/api/agent/generate-images`

## 向后端/研究生确认 Wiki 上传模块

本地当前只看到 `cdc_upload_task` 表、实体和 Mapper，没有看到真正的上传 Controller。

需要确认：

1. “已预留上传接口”具体是哪个接口路径？
   - 是否是 `POST /api/wiki/upload`
   - 代码在哪个文件

2. 当前实现到哪一步？
   - 是否能接收文件
   - 是否能保存文件
   - 是否写入 `cdc_upload_task`
   - 是否能解析文档
   - 是否能清洗分段
   - 是否能写入 Wiki 表

3. 是否有未同步到当前本地/GitHub 的代码？

4. `cdc_upload_task.status` 状态含义
   - 0/1/2/3/4 分别代表什么

5. 第一版支持哪些文件格式？
   - TXT
   - Markdown
   - DOCX
   - PDF
   - XLSX/CSV
   - 图片/OCR

6. 上传后的入库范围
   - 是否写 `wiki_entity`
   - 是否写 `wiki_segment`
   - 是否写 `wiki_rule`
   - 是否写 `wiki_relation`
   - 是否触发 `wiki_segment_embedding`

7. 是否需要人工审核
   - 上传后直接入库
   - 还是先生成待确认结果，再确认入库

8. 更新策略
   - 同名实体覆盖、合并还是跳过
   - 重复片段如何判断
   - 重复上传如何处理
